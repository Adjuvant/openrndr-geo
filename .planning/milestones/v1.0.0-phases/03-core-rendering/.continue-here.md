<context>
**Mental model for Phase 3 execution:**
Core rendering translates geo primitives (Point, LineString, Polygon from Phase 1) + coordinate system transforms (from Phase 2) into visual output using OpenRNDR's drawing system.

The key insight: This is about USER VISUALIZATION CHOICES, not automated styling. CRS is just data origin - the library provides tools, users decide representation (all points red, or color by property, or color by data source).

**Implementation priority:**
1. Style DSL - simple, mutable, zero-allocation
2. Basic rendering - points (3 shapes), lines (dashes, caps/joins), polygons (fill, stroke)
3. Style merging - system defaults + per-feature overrides
4. Data-driven properties - lambda support for size/color driven by feature properties

**Performance target:**
- Real-time animation (points traveling along borders)
- Smooth framerates (no GC pauses from allocation)
- Mutable state for frame-by-frame updates
</context>
