# Milestone v1.1.0: Fix Projection API

**Status:** ✅ SHIPPED 2026-02-26

## Overview

Critical API improvements and bug fixes discovered during v1.0.0 real-world usage. Addresses projection scaling issues, simplifies multi-dataset workflows, and adds animation capabilities.

**Key Theme:** Making the library actually usable for creative geospatial visualization by fixing friction points and reducing boilerplate.

## Phases

### Phase 01: Data Layer

**Goal:** Users can load and access geo data from multiple formats with efficient querying  
**Depends on:** v1.0 foundation  
**Plans:** 4 plans  

Plans:

- [x] 01-01 — Core data model with Feature and Geometry hierarchy
- [x] 01-02 — GeoJSON file loading implementation
- [x] 01-03 — GeoPackage loading with spatial indexing
- [x] 01-04 — Convenience functions for direct feature access

**Details:**
Built foundation with sealed class Geometry (Point, LineString, Polygon, Multi*), lazy bounding boxes, Sequence<Feature> for memory efficiency, and Quadtree spatial index for O(log n) queries. ## Milestone Summary

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Insert Phase 4.1 (CRS-Aware) | Multi-dataset workflows too painful with manual CRS | ✓ Auto-detection eliminates EPSG handling |
| Viewport-relative zoom | Tile-based math (256px) wrong for creative coding | ✓ zoom=0 fits world regardless of viewport |
| Three-tier API | Common case too verbose | ✓ One-liner for simple, full control when needed |
| Use OpenRNDR built-in Easing | Fewer dependencies | ✓ Works well with 15 convenience functions |
| Sealed class for Geometry | Compile-time exhaustiveness | ✓ No runtime errors from missed geometry types |
| Sequence<Feature> | Memory-efficient large datasets | ✓ Handles multi-GB GeoPackages |

### Issues Resolved

- Fixed projection scaling confusion (scale → zoomLevel)
- Fixed fitBounds not actually fitting bounds
- Fixed fitWorldMercator not fitting world
- Fixed MultiPolygon failures on ocean/whole-world data
- Fixed too much boilerplate for common workflows
- Fixed manual CRS handling pain for multi-dataset

- Tier 1 API (drawer.geoJSON()) lacks usage examples
- Redundant else branch in render_BasicRendering.kt
