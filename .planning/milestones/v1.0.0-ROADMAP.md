# Milestone v1.0.0: MVP

**Status:** ✅ SHIPPED 2026-02-22
**Phases:** 1-5 (+ 4.1 inserted)
**Total Plans:** 21

## Overview

Build a Kotlin/OpenRNDR geospatial visualization library that enables creative coders to layer disparate datasets and explore intersections through novel visual forms. The journey starts with data ingestion and coordinate foundations, progresses through rendering and layer composition, and culminates with animation primitives for motion-based visualization.

## Phases

### Phase 1: Data Layer

**Goal**: Users can load and access geo data from multiple formats with efficient querying
**Depends on**: Nothing (first phase)
**Requirements**: DATA-01, DATA-02, DATA-03
**Plans**: 4 plans

Plans:

- [x] 01-01: Create core data model (Bounds, Geometry sealed class, Feature, GeoSource)
- [x] 01-02: Implement GeoJSON file loading
- [x] 01-03: Implement GeoPackage file loading with spatial indexing
- [x] 01-04: Add convenience functions for direct feature access (gap closure from UAT)

**Details:**
- Core data model with sealed class hierarchy for geometry types
- GeoJSON loading via kotlinx.serialization
- GeoPackage loading via geopackage-java with Quadtree spatial index
- Type-safe property access via reified generics
- Lazy Sequence-based iteration for memory efficiency

### Phase 2: Coordinate Systems

**Goal**: Users can transform coordinates between geographic systems and render in basic projections
**Depends on**: Phase 1
**Requirements**: COORD-01, COORD-02, COORD-03
**Plans**: 3 plans

Plans:

- [x] 02-01: Create projection infrastructure (interface, configuration, exceptions, internal implementations)
- [x] 02-02: Implement public projections (Mercator, Equirectangular, BNG) with DSL configuration
- [x] 02-03: Implement screen transformation utilities

**Details:**
- GeoProjection interface with project/unproject/configure/fitWorld methods

## Milestone Summary

**Key Decisions:**

- Use OpenRNDR Vector2 for points (integrates with drawing)
- Sealed class hierarchy enables exhaustive when expressions
- Lazy Sequence-based iteration for memory efficiency
- Use proj4j for CRS transformations (proven Java library)
- Use OpenRNDR built-in Easing (not orx-easing) - fewer dependencies

**Issues Resolved:**

- Fixed Bounds.expandToInclude() for empty bounds (NaN handling)
