# Phase 15-02: Streamlined API Implementation

Implement the two-function API (geoSource/loadGeo) with inline styling DSL and update all examples.

## Overview

This plan implements the streamlined API:
- `loadGeo()` — Auto-magic loader (auto-CRS, auto-projection, auto-fit, caching)
- `geoSource()` — Explicit loader (manual control over all aspects)
- `project()` — Simplified projection helper
- Inline style DSL — `drawer.geo(source) { stroke = BLUE }`

**Hard break:** No backward compatibility. All 26 examples must be updated.

     - Tight fit: 100% viewport (data touches edges)
   - `fun GeoSource.project(projection: GeoProjection): ProjectedGeoSource`
     - Returns wrapper with pre-projected coordinates
   - `fun CachedGeoSource.projectToFit(width: Int, height: Int): GeoProjection`

   - Add overload: `fun Drawer.geo(source: GeoSource, block: Style.() -> Unit)`
     - Creates Style(), applies block, renders with that style
   - Add overload: `fun Drawer.geo(source: GeoSource, projection: GeoProjection, block: Style.() -> Unit)`
   - Keep existing GeoRenderConfig overload for advanced use: `fun Drawer.geo(source: GeoSource, config: GeoRenderConfig.() -> Unit)`
   - Ensure no ambiguity: Style block has no parameters, GeoRenderConfig block has receiver

3. Ensure Style class supports DSL-style configuration:
   - Mutable properties: stroke, fill, strokeWeight, size
   - Default values from StyleDefaults
   - Type-safe builder pattern

4. Default styling (when no block provided):
   - White lines (ColorRGBa.WHITE)
   - Red fills (ColorRGBa.RED)
   - Thin stroke: 1.5
   - Points as circles: radius = 5.0

Example usage:
```kotlin
// Inline style
val data = loadGeo("world.json")

    stroke = ColorRGBa.BLUE
    strokeWeight = 2.0
    fill = ColorRGBa.TRANSPARENT
}

// With explicit projection
val projection = data.projectToFit(width, height)
drawer.geo(data, projection) {
    stroke = ColorRGBa.WHITE
}
```
