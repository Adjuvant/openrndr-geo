# Phase 15-01: Import Structure Reorganization

Reorganize package structure to enable tiered wildcard imports while respecting Kotlin sealed class constraints.

## Overview

This plan establishes the new import structure:
- `geo.*` — Core data types (Geometry, GeoSource, Feature, Bounds, CRS)
- `geo.projection.*` — Projections and transformation utilities
- `geo.render.*` — Rendering extensions (drawer.geo(), Style)
- `geo.animation.*` — Animation utilities (GeoAnimator)
- `geo.layer.*` — Layer utilities (Graticule)

**Critical Constraint:** Geometry sealed class and all implementations (Point, LineString, Polygon, etc.) must remain in the `geo` package root. Kotlin requires sealed class subclasses to be in the same package.
