<context>

The antimeridian splitting algorithm was designed to:
1. Detect edges where |lonDiff| > 180 (raw difference, not normalized)
2. Interpolate crossing latitude at the antimeridian boundary
3. Create new fragments with boundary points as boundaries between fragments
4. Return all fragments without filtering (client code can filter if needed)

The test suite in AntimeridianSplitterTest.kt now all passes, but there's a discrepancy with OptimizedGeometryNormalizerTest. The algorithm produces fragments that may have boundary points as last elements, which differs from what some tests expect (closed rings with first == last).

Next step is to examine the OptimizedGeometryNormalizerTest to understand what it actually needs vs what it expects.
</context>
