<decisions_made>
**Key architectural decision**: This is an OpenRNDR plugin/library, not a pure Kotlin geo library
- Must integrate with OpenRNDR's drawing system
- Use OpenRNDR types directly (e.g., Vector2 for points)
- Library should feel natural to creative coders using OpenRNDR

**API design philosophy**:
- File paths as strings for loading (creative coder-friendly)
- Infix notation DSL for spatial queries (`source.features within bounds`)
- Map-like access for properties (`feature.properties["name"]`)
- Both lazy (sequence) and eager (list) options available

**Error handling approach**:
- Fail-fast on file-level errors (throw exceptions)
- Permissive on feature-level errors (skip with warning)
- Explicit null handling for missing properties
</decisions_made>
