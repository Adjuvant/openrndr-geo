<objective>
Fix zoom level semantics to be viewport-relative (not tile-based) and fix fitWorldMercator to actually fit the world.

**Problem:** The current implementation uses tile pyramid semantics (`scale = 256 * 2^zoom`) which is wrong for creative coding where viewport sizes vary. Users need `zoomLevel = -2` to see the whole world at 800x600.

**Solution:** Make zoom relative to "data bounds fills viewport":
- `zoom=0` = data bounds fills viewport (with optional padding)
- `zoom=1` = 2x zoomed in from baseline
- `zoom=-1` = 2x zoomed out

**Output:**
- Fixed zoom semantics in ProjectionConfig
- Working fitWorldMercator in ProjectionFactory
- Updated examples without TODO workarounds
</objective>

```kotlin
fun fitWorldMercator(width: Double = 800.0, height: Double = 600.0): GeoProjection {
    // Calculate scale that fits world in viewport
    // World width = 2*PI radians, world height = 2*ln(tan(PI/4 + maxLat/2))
    val worldWidth = 2 * PI
    val worldHeight = 2 * ln(tan(PI / 4 + Math.toRadians(MAX_MERCATOR_LAT) / 2))
    
    val scaleX = width / worldWidth
    val scaleY = height / worldHeight
    val scale = min(scaleX, scaleY)
    
    // zoom=0 means world fits viewport, so baseScale = scale
    // Return projection with zoom=0 (which now means world fits)
    return ProjectionMercator(ProjectionConfig(
        width = width,
        height = height,
        center = Vector2(0.0, 0.0),
        zoomLevel = 0.0,  // Now actually fits world!
        bounds = null
    ))
}
```
