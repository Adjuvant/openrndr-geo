Update ViewportCache.getProjectedCoordinates() to handle dirty flag (now that Geometry has the property):

```kotlin
fun getProjectedCoordinates(
    geometry: Geometry,
    viewportState: ViewportState,
    projector: () -> Array<Vector2>
): Array<Vector2> {
    // Clear cache on viewport change
    if (viewportState != currentViewportState) {
        cache.clear()
        currentViewportState = viewportState
    }
    
    val key = CacheKey(viewportState, geometry)
    
    // Check dirty flag and invalidate if needed
    if (geometry.isDirty) {
        cache.remove(key)
        geometry.isDirty = false  // Clear flag after invalidation
    }
    
    // Check size limit
    if (cache.size >= MAX_CACHE_ENTRIES) {
        cache.clear()
    }
    
    return cache.getOrPut(key) { projector() }
}
```

This implements the full dirty flag lifecycle per 12-CONTEXT.md:
1. New geometry starts with isDirty = true
2. First projection: cache miss, stores result, clears flag
3. Geometry modified (future work): markDirty() sets flag
4. Next render: cache sees dirty=true, invalidates entry, clears flag, projects again
