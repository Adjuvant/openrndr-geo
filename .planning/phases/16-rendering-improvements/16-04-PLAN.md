Integrate geometry normalization into the loading pipeline and fix hole rendering in standard rendering paths.

Additionally, standard rendering paths completely ignore interior rings (holes). The existing writePolygonWithHoles() uses Shape(listOf(extContour) + holeContours) with winding enforcement, which does NOT cut holes in OPENRNDR. The correct approach is compound { difference {} } boolean operations.

## Gap Analysis

### Gap 1: Geometry Normalization Not Integrated
Root cause: normalizePolygon() and normalizeMultiPolygon() exist but are never called.

Impact: Ocean.geojson data that crosses antimeridian renders with world-spanning artifacts.

Fix locations:
- parsePolygon() in GeoJSON.kt - must call normalizePolygon() and handle List<Polygon> return
- parseMultiPolygon() in GeoJSON.kt - must call normalizeMultiPolygon()

Key challenge: normalizePolygon() returns List<Polygon> when antimeridian splitting occurs, but parseGeometry() returns single Geometry. Solution: when the list has >1 polygon, wrap in MultiPolygon.

### Gap 2: Holes Not Rendering - Wrong OPENRNDR API
Root cause: writePolygonWithHoles() in PolygonRenderer.kt uses Shape(listOf(extContour) + holeContours) with manual .clockwise/.counterClockwise winding enforcement. This does NOT cut holes in OPENRNDR. The Shape() constructor simply groups contours - it does not perform boolean subtraction.

Correct approach: OPENRNDR compound { difference {} } builder performs explicit boolean subtraction. The exterior is the base shape; each hole contour is converted to a shape and subtracted. compound returns List<Shape>, drawn with drawer.shapes(). No manual winding enforcement needed.

Additionally: DrawerGeoExtensions.kt ignores interior rings entirely in standard rendering paths.

Impact: polygonsWithHole.geojson show solid polygons without hole cutouts.

Fix locations:
- writePolygonWithHoles() in PolygonRenderer.kt - must use compound { difference {} }
- Geometry.renderToDrawer() for Polygon (lines 366-368) - only renders exterior
- Geometry.renderToDrawer() for MultiPolygon (lines 382-386) - only uses poly.exterior
- projectGeometryToArray() (lines 504-507) - only projects exterior coordinates
- renderProjectedCoordinates() (lines 526-548) - only renders exterior from cache

Reference: OPENRNDR guide (guide.openrndr.org/drawing/curvesAndShapes.html) documents compound { difference {} } as the correct boolean operation approach. The shape {} builder example in the guide uses winding for simple cases but compound is more robust and explicit.

        fun writePolygonWithHoles(
            drawer: Drawer,
            exterior: List<Vector2>,
            interiors: List<List<Vector2>>,
            style: Style
        ) {
            if (exterior.size < 3) return

            drawer.fill = style.fill ?: ColorRGBa.WHITE.withAlpha(0.0)
            drawer.stroke = style.stroke ?: ColorRGBa.WHITE
            drawer.strokeWeight = style.strokeWeight
            drawer.lineCap = style.lineCap
            drawer.lineJoin = style.lineJoin

            val extContour = ShapeContour.fromPoints(exterior, closed = true)

            val holeContours = interiors.filter { it.size >= 3 }.map { ring ->
                ShapeContour.fromPoints(ring, closed = true)
            }

            if (holeContours.isEmpty()) {
                drawer.shape(extContour.shape)
            } else {
                // compound { difference {} } performs explicit boolean subtraction.
                // No manual winding enforcement needed - the first shape is the
                // base, all subsequent shapes are subtracted from it.
                // Returns List<Shape>.
                val result = compound {
                    difference {
                        shape(extContour.shape)
                        for (hole in holeContours) {
                            shape(hole.shape)
                        }
                    }
                }
                drawer.shapes(result)
            }
        }

    Key changes from current implementation:
    - Removed .clockwise / .counterClockwise winding enforcement
    - Replaced Shape(listOf(extContour) + holeContours) with compound { difference {} }
    - compound returns List<Shape>, drawn with drawer.shapes() not drawer.shape()
    - Overlapping holes handled correctly by boolean difference
