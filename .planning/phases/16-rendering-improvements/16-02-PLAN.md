       ```kotlin
       val extContour = ShapeContour.fromPoints(exterior, closed = true).clockwise

       val holeContours = interiors.filter { it.size >= 3 }.map { ring ->

       ```
    
    3. Combine into single Shape:
       ```kotlin
       val shape = Shape(listOf(extContour) + holeContours)
       drawer.shape(shape)
       ```
    
    Key changes:
    - Import `org.openrndr.shape.Shape`
    - Use `.clockwise` property on exterior contour
    - Use `.counterClockwise` property on interior contours
    - This ensures correct fill behavior via non-zero winding rule
    - Remove the shape { } builder DSL in favor of Shape(contours) constructor
    
    Rationale from CONTEXT.md:
    - Exterior → `.clockwise` in screen space (positive fill)
    - Interior → `.counterClockwise` in screen space (negative fill = hole)
    - Non-zero winding rule: same direction reinforces, opposite subtracts

    Modify `drawMultiPolygon` in `src/main/kotlin/geo/render/MultiRenderer.kt` to render as single Shape with all contours combined.
    
    Current implementation (per-polygon):
    ```kotlin
    polygonsToRender.forEach { polygon ->
        if (polygon.hasHoles()) {
            val exterior = polygon.exteriorToScreen(projection)
            val interiors = polygon.interiorsToScreen(projection)
            writePolygonWithHoles(drawer, exterior, interiors, style)
        } else {
            val exterior = polygon.exteriorToScreen(projection)
            writePolygon(drawer, exterior, style)
        }
    }
    ```
    
    New implementation (combined Shape):
    ```kotlin
    // Collect all contours from all polygons
    val allContours = mutableListOf<ShapeContour>()
    
    polygonsToRender.forEach { polygon ->
        // Project exterior to screen space
        val screenExterior = polygon.exteriorToScreen(projection)
        // Create clockwise contour (exterior)
        val extContour = ShapeContour.fromPoints(screenExterior, closed = true).clockwise
        allContours.add(extContour)
        
        // Project and add all holes with counter-clockwise winding
        polygon.interiors.forEach { ring ->
            if (ring.size >= 3) {
                val screenRing = ring.map { projection.project(Point(it.x, it.y)) }
                val holeContour = ShapeContour.fromPoints(screenRing, closed = true).counterClockwise
                allContours.add(holeContour)
            }
        }
    }
    
    // Single draw call with all contours
    drawer.shape(Shape(allContours))
    ```
    
    Key implementation details:
    - Import `org.openrndr.shape.Shape` and `org.openrndr.shape.ShapeContour`
    - Use `Polygon.exteriorToScreen()` and `Point.toScreen()` for projection
    - Apply `.clockwise` to all exteriors
    - Apply `.counterClockwise` to all interiors
    - Filter out degenerate rings (< 3 vertices)
    - Single `drawer.shape(Shape(allContours))` call at the end
    
    Benefits per CONTEXT.md:
    - No overdraw at shared boundaries
    - No seams with semi-transparent styles
    - Adjacent polygons merge seamlessly (same winding reinforces)
    - Holes punch through correctly (opposite winding subtracts)
    
    Keep existing parameters:
    - `clampToMercatorBounds: Boolean = true` behavior preserved
    - Style merging with `mergeStyles(StyleDefaults.defaultPolygonStyle, userStyle)`
