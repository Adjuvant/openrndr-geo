### 5. Geometry Normalization Integration
expected: Load a GeoJSON file with a polygon that crosses the antimeridian. Verify that normalizePolygon() is called during parsing by checking that split polygons are promoted to MultiPolygon when the list size > 1. The code should compile and the geometry should be normalized at load time.
result: issue
reported: "FAIL. Call chain confirmed (parseFeature>parseGeometry>parsePolygon>normalizePolygon, normalizeAntimeridian being called, splitAtAntimeridian being called), but wrapping polygons still render 'the long way' across ~359° of longitude. ROOT CAUSE: QGIS uses scanline rasterizer (fills between edge crossings per row) while OPENRNDR traces contour path as vector and fills enclosed shape. For antimeridian-wrapping coordinates, vector path zigzags producing different fill artifacts. SOLUTION: Must normalize coordinates at parse time by splitting rings at ±180 using algorithm that: (1) detects edges with |Δlon| > 180, (2) splits into pos/neg longitude buffers, (3) interpolates boundary points at ±180, (4) promotes crossing Polygons to MultiPolygons, (5) splits holes same way and assigns fragments to matching exterior. Pre-split data (ocean.geojson touching exactly ±180.0) should pass unchanged. Test file: uat/polygon-bug-fixing.kt with data/geo/test-antimeridian-crossing.geojson"
severity: blocker
