  <files>src/main/kotlin/geo/render/geometry/AntimeridianSplitter.kt</files>
  <action>
    Create a new file at `src/main/kotlin/geo/render/geometry/AntimeridianSplitter.kt` that provides antimeridian-aware polygon processing.
    
    The file should implement:
    
    1. `crossesAntimeridian(ring: List<Vector2>): Boolean`
       - Detect if any adjacent coordinate pair has longitude jump > 180°
       - Use: `kotlin.math.abs(next.x - current.x) > 180.0`
    
    2. `interpolateAntimeridianCrossing(p1: Vector2, p2: Vector2): Double`
       - Calculate latitude where segment crosses ±180° boundary
       - Algorithm: Calculate how far along the segment the crossing occurs
       - For crossing from + to -: deltaLon = (180.0 - p1.x) + (p2.x + 180.0)
       - For crossing from - to +: deltaLon = (-180.0 - p1.x) + (p2.x - 180.0)
       - Interpolate latitude linearly: p1.y + t * (p2.y - p1.y)
    
    3. `splitAtAntimeridian(ring: List<Vector2>): List<List<Vector2>>`
       - Split a ring crossing antimeridian into separate rings
       - Walk coordinate pairs, detect crossings
       - At crossing: interpolate vertex at boundary, split ring
       - Start new ring from opposite side of boundary
       - Return list of closed rings (each ring already closed by algorithm)
    
    4. `normalizeLongitude(lng: Double): Double` (if needed, or use existing from Geometry.kt)
       - Ensure longitude is in [-180, 180] range
    
    Key implementation details:
    - Import: `org.openrndr.math.Vector2`
    - Rings are closed (last vertex connects to first)
    - Handle edge cases: ring that crosses multiple times, ring entirely on one side
    - Make all functions `internal` visibility for render package use
    
    Testing via TDD (Wave 0 test required):

    - Tests: simple crossing, multiple crossings, no crossing, edge at exactly ±180°

    4. `normalizePolygonWinding(
           exterior: List<Vector2>,
           interiors: List<List<Vector2>>

       - Normalize exterior to clockwise (for screen space fill)
       - Normalize all interiors to counter-clockwise (for holes)
       - Return normalized exterior and interiors
    
    Key implementation details:
